# RAG AI写作工作流

本文档用于把 `raw_materials` 中的文章沉淀为可复用的 AI 写作参考库，并形成一套稳定、可迭代的写作工作流。

文档默认编码要求：

- 全部文件统一使用 `UTF-8`
- Markdown 文件使用 `UTF-8`
- JSON、CSV、YAML、TXT 一律使用 `UTF-8`
- 不要使用 `ANSI`、`GBK`、`UTF-8 with BOM`
- Windows 批量处理前先检查编码，避免中文标签、标题、摘要乱码

## 1. 总体目标

最终希望实现三层能力：

1. 沉淀一套跨文章通用的核心写作 skills，作为 AI 的总纲领。
2. 将历史文章按行业、文体、风格、主题、受众、写作意图等维度结构化归类，支持检索、推荐和打分。
3. 在用户发起一次具体写作任务时，自动找到最适合的参考文章，提炼出本次写作最关键的写法要点，作为单篇任务的动态参考缓存。

对应到系统层面，可以拆成三层：

- `Global Writing Skills Layer`
- `Reference Retrieval And Ranking Layer`
- `Task-Specific Writing Cache Layer`

- `prompt_bundle.md` 是最终写作时实际喂给 AI 的完整上下文包，里面包含总纲、单篇任务摘要和精选参考原文。

## 3. 三层能力设计

### 3.1 全局总纲：核心写作 skills

这一层的目标不是记住具体事实，而是沉淀“写作方法论”。

这部分应该回答的问题：

- 这批文章在选题上有什么共同规律
- 标题通常如何制造信息密度和吸引力
- 开头常见的起势方式是什么
- 论证展开如何安排节奏
- 如何在观点、案例、数据、金句、提问之间切换
- 如何做结尾收束和行动号召
- 什么语气最符合你的目标写作风格
- 哪些写法是高频有效的，哪些写法应该避免

建议把全局总纲拆成以下模块：

1. 选题原则
2. 标题原则
3. 开篇原则
4. 结构原则
5. 论证原则
6. 风格原则
7. 信息组织原则
8. 结尾原则
9. 禁忌清单
10. 质量评分标准

`core_writing_skills.md` 的作用：

- 作为所有写作任务的系统提示词基础
- 无论写哪个行业、哪种文体，都先加载这一层
- 它决定文章整体调性、逻辑密度、信息组织方式和品牌一致性

### 3.2 分类检索层：按类归纳、推荐和打分

这一层负责“找到适合参考的历史文章”。

每篇文章至少应有以下标签字段：

- `title`：文章标题
- `year`：年份
- `industry`：行业，如消费、科技、制造、金融、医疗、教育、组织管理
- `topic`：主题，如战略、创新、领导力、营销、组织、人力、AI、ESG
- `genre`：文体，如评论、案例分析、人物专访、趋势解读、活动预告、书摘、观点短文
- `style`：风格，如理性分析、启发式、访谈型、故事型、学术转译型、媒体报道型
- `audience`：受众，如企业管理者、创业者、职场人、研究者、公众读者
- `intent`：写作意图，如解释概念、输出观点、拆解案例、建立认知、引导报名、推广活动
- `tone`：语气，如冷静、犀利、克制、洞察、鼓动、温和
- `structure_pattern`：结构模板，如“问题-分析-建议”“案例-洞察-方法”“现象-原因-趋势”
- `quality_score`：综合质量分
- `reference_value_score`：作为参考样本的价值分

这里的重点不是标签越多越好，而是标签必须能支持检索和排序。

### 3.3 单篇任务缓存层：本次写作的动态参考包

这一层是最直接提升产出质量的关键层。

当用户要写一篇新文章时，系统先接收写作任务描述，例如：

- 行业：AI 医疗
- 文体：案例分析
- 风格：理性但有传播性
- 目标受众：企业管理层
- 文章目的：解释趋势并给出判断

然后执行：

1. 用任务描述去检索历史文章。
2. 召回最相近的 `top 20`。
3. 再重排出最值得参考的 `top 3` 到 `top 5`。
4. 从这几篇里提炼本次写作的关键方法。
5. 生成一份短小但高价值的 `writing_insights.md`。
6. 把 `core_writing_skills.md`、`writing_insights.md` 和筛选后的精选参考原文一起喂给 AI。

这一层不是永久知识，而是“只服务本次写作”的任务缓存。

## 4. 推荐的数据处理流程

## 4.1 第一步：原始文章清洗

目标：让每篇文章都成为干净、可读、可切分、可标注的文本。

清洗建议：

- 去掉无意义占位文本
- 去掉“分享图片”这类无正文价值文件，或者打上低价值标签
- 统一换行和空格
- 去掉重复标题、重复尾注、无意义来源信息
- 如果正文太短，标记为 `low_content`
- 如果明显属于活动通知、报名通知、直播预告、工作坊宣传，则直接删除，不进入 RAG 参考库

清洗后每篇文章至少应保留：

- 标题
- 正文
- 日期
- 来源路径

## 4.2 第二步：生成单篇摘要与标签

针对每篇文章，调用 AI 生成一份结构化摘要。

建议输出字段：

```json
{
  "title": "",
  "date": "",
  "year": "",
  "industry": [],
  "topic": [],
  "genre": "",
  "style": [],
  "audience": [],
  "intent": [],
  "tone": [],
  "structure_pattern": "",
  "core_argument": "",
  "key_points": [],
  "opening_pattern": "",
  "ending_pattern": "",
  "notable_phrases": [],
  "quality_score": 0,
  "reference_value_score": 0,
  "summary_200": "",
  "summary_500": ""
}
```

这里要特别区分两个分数：

- `quality_score`：文章本身写得好不好
- `reference_value_score`：这篇文章适不适合被拿来模仿和参考

有些文章信息价值高，但不一定适合做写作模板。

## 4.3 第三步：抽取全局写作规律

在完成一批文章摘要后，再做一次“跨文档分析”，总结共性规律，产出 `core_writing_skills.md`。

抽取角度建议：

- 高分文章最常见的标题模式
- 高分文章最常见的开头方式
- 高分文章的段落长度和节奏
- 高频论证动作，例如定义问题、提出反常识、引入案例、上升到趋势
- 高频转折语和收束语
- 常见失败模式，例如空泛、啰嗦、观点不够尖锐、结构松散

这是整个库最重要的长期资产。

## 6. 检索与推荐机制

建议使用“两段式检索 + 一段式重排”。

### 6.1 第一段：粗召回

目标是先把可能相关的文章找出来。

可以同时使用：

- 稀疏检索：关键词、BM25、标题命中、标签过滤
- 稠密检索：向量召回，理解语义相似度

召回输入可包括：

- 用户给出的标题意图
- 行业
- 文体
- 风格
- 目标受众
- 写作目的
- 用户提供的提纲或关键词

### 6.2 第二段：重排

对召回结果重新打分，得到最值得参考的少量文章。

建议综合以下信号：

- 主题相似度
- 文体匹配度
- 风格匹配度
- 结构相似度
- 标题可借鉴度
- 开头可借鉴度
- 高质量分加权
- 高参考价值分加权

一个可落地的重排公式示意：

```text
final_score =
0.30 * semantic_similarity +
0.20 * genre_match +
0.15 * style_match +
0.15 * topic_match +
0.10 * quality_score +
0.10 * reference_value_score
```

### 6.3 第三段：生成单篇参考摘要

最终不要把整批文章全文都塞给 AI。

只保留：

- 最佳参考文章 `3` 到 `5` 篇
- 每篇文章的核心可借鉴点
- 本次任务的综合写法建议
- 以及这几篇精选文章的原文全文

输出为 `writing_insights.md`。

## 7. 单篇任务工作流

下面是每次实际写作时建议执行的标准流程。

### 7.1 输入

用户提交一份任务卡：

```text
写作主题：
目标标题方向：
行业：
文体：
风格：
目标受众：
文章目标：
是否需要案例：
是否需要数据：
期望字数：
是否有既定观点：
```

### 7.2 系统执行

1. 解析任务卡，提取结构化条件。
2. 调用标签库和向量库检索候选文章。
3. 根据排序规则选出 `top 3` 到 `top 5`。
4. 对这几篇文章做“任务相关提炼”。
5. 生成本次 `writing_insights.md`。
6. 加载 `core_writing_skills.md`。
7. 把 `core_writing_skills.md`、`writing_insights.md` 和精选参考原文组装为 `prompt_bundle.md`。
8. 用 `prompt_bundle.md` 驱动 AI 写作。

### 7.3 输出

本次写作至少应该输出三份内容：

1. 参考文章列表
2. 本次写作关键点摘要
3. 最终文章草稿

## 8. 写作提示词分层设计

建议把提示词拆成三层，避免一个超长提示词不可维护。

### 8.1 第一层：系统总纲提示词

来源：`core_writing_skills.md`

作用：

- 约束整体风格
- 约束逻辑与结构
- 约束写法禁忌
- 保持品牌一致性

### 8.2 第二层：任务参考提示词

来源：`writing_insights.md`

作用：

- 告诉 AI 本次最应该借鉴哪些文章
- 告诉 AI 本次应学习哪些标题、开头、论证动作、结尾方式
- 告诉 AI 此次最重要的写法重点
- 但不替代精选参考原文，只负责提炼

### 8.3 第三层：用户任务提示词

来源：用户输入

作用：

- 明确主题、目标、字数、读者、约束条件
- 要求输出格式

## 10. `core_writing_skills.md` 模板

建议长期维护成下面这种结构：

```md
# 核心写作总纲

## 一、写作目标
- 文章应服务于认知提升、观点表达和高质量传播

## 二、标题原则
- 标题要有明确主题、信息密度和阅读抓手

## 三、开头原则
- 开头优先建立问题感、冲突感或认知反差

## 四、结构原则
- 全文应有稳定主线，避免段落并列堆砌

## 五、论证原则
- 每个核心观点都应有解释、案例、推演或证据支撑

## 六、风格原则
- 保持理性、克制、清晰，不写空洞口号

## 七、结尾原则
- 结尾要完成收束、提升或行动引导

## 八、禁忌
- 避免空泛大词
- 避免逻辑跳跃
- 避免结构松散
- 避免套话化表达
```

## 11. 评分机制设计

为了让系统能自动推荐“最值得参考”的文章，需要一套稳定评分标准。

建议总分 `100`，可拆成：

- `20` 分：标题质量
- `20` 分：开头吸引力
- `20` 分：结构清晰度
- `15` 分：观点深度
- `10` 分：案例或证据支撑
- `10` 分：语言表现力
- `5` 分：结尾收束力

另设一个独立的参考价值分：

- 是否适合模仿
- 是否结构清晰
- 是否有可复用模板
- 是否具有代表性

建议把“高质量”和“高参考性”分开保存，不要混成一个指标。

## 12. 人工校准流程

不要完全依赖 AI 自动打标签和打分，前期一定要人工校准。

建议做法：

1. 先抽样 `100` 篇文章做人工校准。
2. 修正行业、文体、风格和评分标准。
3. 固化标签体系后，再批量处理剩余文章。
4. 每月复查一批推荐结果，修正误差。

人工校准重点看三件事：

- 标签是否稳定
- 推荐结果是否真的像
- AI提炼出来的写法要点是否对写作有帮助

## 13. 实际执行顺序

建议按下面的顺序推进，而不是一步到位。

### 阶段一：建库

1. 清洗原始文章
2. 统一命名和编码
3. 建立文章总台账

### 阶段二：结构化

1. 给每篇文章打标签
2. 生成摘要
3. 生成评分

### 阶段三：抽取总纲

1. 分析高分文章共性
2. 形成 `core_writing_skills.md`
3. 反复修订

### 阶段四：上线任务流

1. 接收写作需求
2. 自动检索与重排
3. 自动生成 `writing_insights.md`
4. 驱动 AI 完成初稿

### 阶段五：闭环优化

1. 记录哪些参考文章最有效
2. 记录哪些提示词组合效果最好
3. 记录哪些生成结果质量高
4. 反向优化标签、评分和总纲

## 14. 你以后实际使用时的最简流程

每次写文章，按这个顺序即可：

1. 写一份任务卡。
2. 从库里召回相关文章。
3. 挑出最优参考 `3` 到 `5` 篇。
4. 提炼本次关键写法，生成 `writing_insights.md`。
5. 把 `core_writing_skills.md`、`writing_insights.md` 和精选参考原文组装为 `prompt_bundle.md`。
6. 把 `prompt_bundle.md` 交给 AI。
7. 人工审一轮，再把优质结果回写进库中。

## 15. 关键原则

- 全局总纲解决“长期稳定风格”。
- 分类检索解决“找到像的文章”。
- 单篇缓存解决“本次到底该怎么写”。
- 精选参考原文要喂给模型，但只能喂经过筛选后的 `3` 到 `5` 篇，不能整库直塞。
- 写作库不是知识库的简单堆积，而是“可复用写法”的结构化资产。

## 17. 模型建议

当前实现建议优先使用 Gemini Flash Lite 作为批量分析、标签生成、重排和任务摘要模型。

原因：

- 成本更适合大规模批处理
- 速度更适合几千篇文章的结构化分析
- 足够支持标签抽取、摘要生成、重排和写作提示整理

如果后续需要更强的深度归纳能力，可以在生成 `core_writing_skills.md` 或生成最终文章时，再切换到更强模型。

## 16. 下一步建议

如果要真正落地，下一步最值得做的是四件事：

1. 先建立 `taxonomy.yaml`，固定标签体系。
2. 批量生成 `article_catalog.csv` 和 `article_tags.jsonl`。
3. 从高分文章中提炼第一版 `core_writing_skills.md`。
4. 设计一个标准化的“写作任务卡”输入模板。

等这四步完成后，这个库就可以开始承担 AI 写作参考库的角色。
