# 项目全景与成果说明

项目名称：`Writing Workspace`

定位：`AI 驱动的商业文章写作工作台`

## 要解决的问题

- 商业文章的前期资料分散，研究成本高。
- 长文写作容易在结构阶段返工。
- 模型单次直出长度有限，长文稳定性差。
- 终稿常见问题集中在重复、空泛、AI 腔和节奏失衡。

## 当前解决方案

系统把写作拆成一条稳定链路：

`选题 -> 搜索研究 -> 整理资料 -> 选择讨论方向 -> 审阅大纲 -> 分段成文 -> 审查 -> 导出`

核心改动有三点：

1. 主产物从旧的教学型文档切换为商业文章。
2. TN / 讨论指南保留，但改为可选产物。
3. 长文改为 chunk-based 写作，每轮约 `1500` 字，再续写拼接。

## 当前交付物

- 正文终稿
- `writing_insights.md`
- `evidence_cards.md`
- `outline.md`
- `critique.md`
- 可选 TN / 讨论指南
- 正文 Markdown / PDF

## 技术实现摘要

- React + TypeScript + Vite
- `@google/genai`
- Markdown prompt 资产运行时桥接
- 本地 RAG 资产脚本用于离线分析和写作规则沉淀
