# 工作流 SOP 与生产规范

## 标准流程

1. 输入主题与任务配置。
2. 上传补充资料。
3. 运行搜索研究。
4. 从候选方向里选择一个讨论方向。
5. 审阅并修改大纲。
6. 启动分段写作。
7. 查看审稿意见并生成终稿。
8. 按需生成 TN。
9. 导出 Markdown / PDF。

## 生产规范

- 所有任务先定方向，再定大纲，不跳过结构阶段。
- 长文默认按 chunk 写作，不要求模型一次写完。
- 单轮目标字数建议保持在 `1200-1500`。
- TN 仅在确有需要时启用。
- 终稿前必须经过一次审稿或终稿清洗。

## 质量门槛

- 标题不能空泛。
- 开头必须快速进入主题。
- 中段要有证据和论证动作。
- 结尾必须完成收束，不做空泛喊话。
- 避免常见 AI 套话和装饰性符号。
