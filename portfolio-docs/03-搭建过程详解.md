# 搭建过程详解

## 改造目标

这次改造不是推翻原有流程骨架，而是保留原来“研究 -> 规划 -> 写作 -> 审查”的产品路径，彻底替换领域语义和产物结构。

## 核心改造点

### 1. 类型层

- `objective` 改为 `direction`
- `framework` 改为 `outline`
- 旧的正文字段改为 `articleContent`
- `teachingNotes` 保留，但由必选改为可选

### 2. prompt 资产层

通过 `?raw` 导入项目内的 Markdown prompt：

- `core_writing_skills.md`
- `universal_prompt.md`
- `ai_writing_workflow.md`
- `task_brief_template.md`

这样写作规则不再散落在 TS 大字符串里。

### 3. 服务层

服务层被改成：

- 搜索研究
- 讨论方向生成
- 大纲生成
- `writing_insights.md` 生成
- `evidence_cards.md` 生成
- chunk plan 生成
- 分段写作
- 审稿
- 终稿改写
- 可选 TN 生成

### 4. 界面层

新的主组件包括：

- `DirectionSelection`
- `OutlineReview`
- `ArticleViewer`
- `WritingCopilot`

## 长文机制

长文不再依赖单次直出，而是：

1. 先生成完整大纲。
2. 再生成 chunk plan。
3. 每轮只写本轮章节。
4. 全部 chunk 合并后再统一审稿和收束。
