## 三、最早阶段完成的认知梳理

在真正开工前，先对项目里“最高层人格”和“核心 prompt”做了完整梳理。

### 3.1 运行时实际生效的人格壳

结论是：前端运行时真正生效的最高层人格来自 `services/geminiService.ts` 中的 `buildSystemInstruction(role)`，而不是设计文档里提到的所有 Markdown prompt 资产。

仓库设计层原本的核心 prompt 体系主要来自：

- `rag_assets/global/universal_prompt.md`

## 十四、仍需知情的风险

虽然当前目标已完成，但仍有三个现实风险需要保留在交接文档里。

### 14.1 软文识别存在边界误差

原因：

1. 本质仍是模型判别
2. 商业媒体稿件中“采访稿”和“带品牌背书的报道”天然存在灰区

当前处理策略是“宁可先打标签隔离，也不让其污染高价值参考库”。

### 14.2 benchmark 题集仍偏通用

目前的人格评测题集已经可用，但 LatePost 风格如果要继续打磨，后续最好增加更贴近：

1. 独家报道
2. 公司组织调整
3. 行业终局判断
4. 人物与企业深描

这些任务的专属 benchmark。

### 14.3 运行时模型选择未收口

这是按你的要求保留的结果：

1. 运行时用户仍可按当前产品逻辑选模型
2. 因此“构建期只用 Flash”与“运行期模型选择自由”是两个独立系统

这不是 bug，但需要在后续文档和产品说明中持续强调，避免混淆。

## 十五、后续建议

如果后续继续迭代，建议优先做以下四件事。

1. 给 LatePost 增补更贴近报道写作的 benchmark task。
2. 把软文标签进一步细分成“品牌合作稿 / 高管口径稿 / 新品发布稿 / 招商传播稿”。
3. 给每个风格库增加独立的反模式黑名单。
4. 在前端展示当前风格库的人格版本、最近一次进化时间和评测结果。
