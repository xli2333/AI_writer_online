# 06 编码规范与乱码处理指南

最后更新：`2026-03-04`  
目标：确保中文文档、JSON、代码在 Windows/macOS/Linux 一致可读。

## 1. 强制标准

- 文档与配置文件统一使用 `UTF-8`。
- 推荐无 BOM（除非下游工具明确要求 BOM）。
- 提交前至少抽查一次中文段落显示效果。

## 2. 编辑器设置建议（VS Code）

在工作区配置：

```json
{
  "files.encoding": "utf8",
  "files.autoGuessEncoding": false,
  "files.eol": "\n",
  "editor.renderControlCharacters": true
}
```

说明：

- `autoGuessEncoding=false` 可以减少“误判编码后保存”的风险。
- `renderControlCharacters=true` 有助于发现隐藏异常字符。

## 3. Windows 终端设置建议

### CMD

```bat
chcp 65001
```

### PowerShell（临时）

```powershell
[Console]::InputEncoding = [System.Text.Encoding]::UTF8
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
```

## 4. Git 编码建议

```bash
git config --global i18n.commitEncoding utf-8
git config --global i18n.logOutputEncoding utf-8
git config --global core.quotepath false
```

说明：

- `core.quotepath=false` 能避免中文文件名被转义显示成八进制序列。

## 5. 快速检测与修复（本目录脚本）

### 5.1 检查 UTF-8 状态

```powershell
powershell -ExecutionPolicy Bypass -File .\portfolio-docs\scripts\Check-Utf8.ps1 -Root .\portfolio-docs
```

输出会标识：

- `OK`：可被严格 UTF-8 解码。
- `WARN`：文本包含替换字符 `U+FFFD`（可能已发生乱码）。
- `ERROR`：无法按 UTF-8 严格解码。

### 5.2 批量转为 UTF-8（谨慎执行）

```powershell
powershell -ExecutionPolicy Bypass -File .\portfolio-docs\scripts\Convert-ToUtf8.ps1 -Root .\portfolio-docs -SourceEncoding gb18030 -WhatIf
```

先用 `-WhatIf` 预演，确认后去掉 `-WhatIf` 再实际写回。

## 6. 常见乱码场景与处理

### 场景 A：终端看源码乱码，但浏览器显示正常

- 原因：终端编码页非 UTF-8。
- 处理：切 `chcp 65001` 或 PowerShell UTF-8 输出后重试。

### 场景 B：文件保存后中文整体异常

- 原因：编辑器按错误编码重写了文件。
- 处理：从 Git 恢复正确版本，重新以 UTF-8 打开并保存。

### 场景 C：路径中含中文导致脚本失败

- 原因：工具链对中文路径兼容不一致。
- 处理：截图和文档资产采用 ASCII 文件名（本项目已落地 Uxx 命名）。

## 7. 发布前编码检查清单

1. Markdown 与 JSON 均通过 UTF-8 检查。
2. 中文截图文件名已标准化（Uxx-*.png）。
3. 文档中无 `U+FFFD` 替换字符。
4. Windows 与 macOS 至少各验证一次可读性（若具备条件）。
