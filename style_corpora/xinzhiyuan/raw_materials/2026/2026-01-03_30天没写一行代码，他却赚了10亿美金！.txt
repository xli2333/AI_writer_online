6个月，10亿美元收入，这就是Claude Code创下的奇迹。就在刚刚，自曝靠Claude Code让AI 100%写代码的工程师，大方揭秘了自己的配置！众多大佬预言：四天工作制真来了。

AI

一个工程师的「副业」

10亿美元
ARR

年度经常性收入

你肯定已经猜到了，它就是Claude Code。
而Claude Code之父已经承认，过去30天里，自己100%的代码，都是靠这个AI写的！
作为Claude Code项目的开发负责人，Boris Cherny可以说是Claude Code技术的灵魂人物。他曾被Cursor开发商 Anysphere重金挖走，然后又被Anthropic闪电速度挖回。
上线半年，收入破10亿
就在25年12月初，AI编程圈炸锅了。

宣布，旗下AI编码神器
Claude Code

6个月，
就创造了近

，堪称AI编程类工具史上最亮眼纪录，直接一条腿跑赢很多大厂半年成绩！
同时，它还完成了令人咋舌的
首笔战略收购——吞下开发者神器Bun。这就意味着，
AI
编码工具已经进入了中后台的基础设施时代，企业的付费市场已经打开！
Claude Code的神奇增长速度，背后有一个关键秘密： 它不是简单的「代码补全器」，而是一个
AI
数字「码农同事」。
一般的AI工具最多也就是补代码片段、解释bug，但Claude Code的目标是：能理解整个项目上下文，自动设计、生成、测试代码，并且与真实工作流程深度融合！
这就意味着：无论是写功能、调试、打包，AI都能自动完成，可以直接在终端/IDE 里一条命令「召唤」，而且几乎不需要手工写冗余代码。
可以说，它已经是一个能和你一起独立写项目的
AI

6个月

快10亿美元

这个数字，比很多
传统软件老牌公司的全年营收还要高！
Claude Code之父，手把手亲授
自从Claude Code之父Boris Cherny承认自己的代码100%由Claude Code写成，AI社区大为震惊：这是怎么做到的？
就在刚刚，Boris大方揭秘了自己的配置！
出乎意料的是，这个配置竟然如此简单。
可以说，Claude Code开箱即用，效果很好，所以Boris Cherny很少对它进行自定义设置。
具体步骤如下。
1.在终端中并行运行5个Claude。
2.在Claude.ai/code上，也同时运行5-10个Claude。
通过终端编码时，他会经常会将本地会话交给网络，或者手动在Chrome中启动会话，有时还会来回传送。
每天早上和白天，他都会从自己手机上（通过Claude iOS应用）启动几个会话，然后会检查它们。
然后，他会用Claude Opus 4.5来思考。可以说，这是他用过最好的编程模型，虽然它比Sonnet更大、更慢，但是因为它需要的引导更少，在工具使用上也更好，所以在小模型中，它几乎总是最快的。
Boris的团队会共享一个CLAUDE.md用于 Claude Code仓库。他们会将其提交到git，整个团队每周贡献多次。
每次他们看到Claude做错事时，就会将其添加到CLAUDE.md ，这样Claude就知道，下次不能这样。
代码审查期间，他常常在同事的PR上标记@.claude，从而将某些内容添加到CLAUDE.md，作为PR的一部分。他们会使用Claude Code Github action (/install-github-action) ，来实现这一点。
大多数会话以Plan模式启动。如果目标是写一个Pull Request，他就会使用Plan模式，并与Claude来回沟通，直到满意为止。
然后，他会切换到自动接受编辑模式，Claude通常可以一次性完成。
一个好的计划，真的非常重要。
Boris说，自己会使用斜杠命令来处理每天多次执行的「内循环」工作流。这样一来，就不用一次次重复写提示词，而且
Claude也可以直接使用这些工作流
。
例如，Claude和他每天都会使用/commit-push-pr这个斜杠命令数十次。
这个命令里会直接内嵌bash脚本，提前计算好
git status
以及一些其他信息，这样命令执行得更快，也避免了和模型之间来回确认、反复沟通。
Boris会经常用到几个
子代理（subagents）。
比如
code-simplifier
，在Claude完成代码后负责对代码进行简化；
verify-app
则有一套非常详细的说明，用来对Claude Code进行端到端测试，等等。
和斜杠命令类似，Boris把子代理看作是对
大多数
PR
中最常见工作流的自动化封装
，会把那些重复、固定的流程交给它们。
他们团队会经常使用
PostToolUse hook，
来对Claude生成的代码进行格式化。
Claude通常一开始就能生成格式很规范的代码，而这个hook主要负责补上
最后那10% 的细节
，从而避免后续在CI中因为格式问题而报错。
他不会使用
--dangerously-skip-permissions
。相反，他会通过
/permissions
预先放行那些能确认在当前环境中是安全的常用bash命令，从而避免频繁弹出不必要的权限提示。
Boris表示，Claude Code已经替他使用所有他的工具。
它经常会搜索信息、通过MCP服务器在Slack上发消息，还会用
bq
CLI 运行BigQuery 查询来回答分析类问题，或者从Sentry拉取错误日志等等。
对于运行时间特别长的任务，他通常会采用以下几种方式之一：
（a）在任务完成后，提示Claude使用一个后台agent来校验自己的工作； （b）使用agent的
Stop hook
，用更确定、可控的方式来完成校验；
（c）或者使用
ralph-wiggum插件。
最后，一个很重要的建议就是：
想要把Claude Code的效果发挥到极致，最重要的一点就是——给Claude一个验证自己工作的方式。
一旦Claude拥有这样的反馈闭环，最终产出的质量通常能提升
2～3倍
。
Boris的X发布后，大量专业人士在帖子下方表示赞叹，而且提出了不少细节问题。
2026，人类离「只工作四天」竟如此之近？
说到这里，就不得不提最近大佬们对于人类未来工作的预测了。
2026年伊始，一场关于
「未来工作模式革命」
的大讨论正在兴起。
曾被视为理想主义者的「四天工作制」，如今正被世界上最有影响力的商业领袖们认真讨论，而且越来越多人认为：
这不再是乌托邦，而是很有可能成为现实！
比尔盖茨认为，未来人类可能每周只需要工作2到3天。
摩根大通的
Jamie Dimon
预测，下一代人将活到100岁且没有癌症，每周仅需工作
3.5天
。
黄仁勋认为，4天工作制是最有可能的。
马斯克的预测则最为激进：他认为未来10到20年内，工作将变成可选项，未来将是
「全民高收入」
且没有贫困的世界。
而且，这场变革并非空想。已经有多国开始试点四天工作制，结果显示，工作效率不降反升，人类员工压力降低，满意度上涨。
然而，这场变革也不是毫无代价的。人们可能会被AI迅速替代，低技能的岗位转型压力巨大，而且失业风险和社会支持体系也需要改革。
然而，技术进步从不会因为恐惧而停下脚步。
可以肯定的是，我们正处在一个劳动方式被彻底改写的时代。
